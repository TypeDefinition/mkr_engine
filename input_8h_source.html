3 /**
4  \file input.h
5  \brief
6  When making games, for portability purposes, as well as our own sanity, we do not want to hardcode physical buttons to specific actions in our games.
7  We should use a layer of abstraction. This is where our input system comes in.\n
8 
9  input_action (32bit unsigned integer) -
10  An input action is the name we give to our actions. Examples can be JUMP, WALK or SHOOT. This is defined by the user.
11 
12  input_mask (16bit input_context + 16bit controller_index + 32bit keycode) -
13  A 64bit mask made by combining an input_context, a controller_index and a keycode.
14  To register a control, we will register with an input handler a input_action and a input_mask.
15  This means that whenever the conditions of the input_mask is met (correct context, controller and physical button), an event representing input_action will be dispatched.
16 
17  input_context (16bit unsigned integer) -
18  The controls of a game might differ depending on what is happening.
19  Contexts is the name given to the scenario the game or player is currently in. Examples can be WALKING, DRIVING, COMBAT, MAIN_MENU, PAUSE_MENU.
20  For example during the context COMBAT, left-click may mean PUNCH, but during the context DRIVING, left-click may mean HONK.
21  This allows us to make the same physical controls do different things in different situations.
22  We can represent these contexts using input_context. As input_context has 16bits, we can have up to 16 contexts.
23 
24  controller_index (16bit unsigned integer) -
25  It is possible to have multiple controllers active at the same time, such as in a couch co-op game.
26  Thus, controller_index allows us to specify which controller we want to trigger which actions.
27  For example in a couch co-op game, we want inputs from controller_index_0 to control player 1, and controller_index_1 to control player 2.
28  We can represent these controllers using controller_index. As controller_index has 16bits, we can have up to 16 controllers.
29 
30  keycode -
31  This is a 32bit unsigned integer that represents the physical key being pressed. For the full list of available keycodes, refer to keycode.h.
32 */
